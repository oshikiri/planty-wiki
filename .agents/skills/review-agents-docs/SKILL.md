# Review Agents Docs

`AGENTS.md` を対象に、運用で再現できるルールへ整える。

## Workflow

1. `AGENTS.md` を読み、既存ルールの重複と矛盾を把握する。
2. `references/agents-writing-guide.md` を読み、このスキルで使う参照資料とレビュー観点を確認する。
3. 優先度順で問題を抽出する。
4. 小さい差分で文書を修正する。
5. 同一テーマの記述を全検索し、重複と矛盾がないことを確認する。
6. 変更内容と残リスクを報告する。

## Review Focus

- 全作業で常に効く上位ルールだけが書かれているかを確認する。
- 各ルールが命令文で書かれ、実行者の行動が即時に決まるかを確認する。
- 長文や曖昧語が残っていないかを確認する。
- `docs/coding-standards.md` と重複する詳細を抱え込んでいないかを確認する。
- 追加または変更したルールに根拠を添えられるかを確認する。
- `AGENTS` の必須記載項目が欠けていないかを確認する。

## Edit Rules

- 一つの箇条書きには一つの要求だけを書く。
- 禁止事項には対象と条件を明示する。
- 重複するルールは一か所へ集約し、必要なら参照を置く。
- 背景説明が長くなる場合は別文書へ分離する。

## Reference

- 詳細ガイドは `references/agents-writing-guide.md` を参照する。
