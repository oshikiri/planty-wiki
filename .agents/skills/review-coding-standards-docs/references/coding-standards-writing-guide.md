# coding-standards.md 運用ガイド

## 目的
- このガイドは `docs/coding-standards.md` を判定可能で保守しやすい規約に保つために使う。

## 書き方

### ルールを検証可能に定義する
- 各ルールは第三者が同じ結論で違反判定できる粒度で書く。
- 各ルールには適用範囲を明示する。
- 例外が必要なルールには例外条件を明示する。

### ルールを一文一義で定義する
- 一つの箇条書きには一つの要求だけを書く。
- 複合要求は分割して独立した行へ書く。
- `必須` と `推奨` を分けて書く。

### ルールに理由と例を付ける
- 重要なルールには理由を一文で添える。
- 誤解しやすいルールには最小の具体例を添える。
- 仕様依存のルールには参照先文書を併記する。

### 章構成を固定して保守しやすくする
- 言語別またはレイヤ別に章を分ける。
- 章名は実装単位に合わせて具体化する。
- アーキテクチャ制約は依存方向と禁止 import を明記する。

## 更新チェック
- 既存ルールとの重複と矛盾がないことを確認する。
- 新規ルールの適用範囲と例外条件が明示されていることを確認する。
